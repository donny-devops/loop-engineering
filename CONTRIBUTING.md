# Contributing

## Ground Rules

- No code changes in week one; triage and observe only.
- All changes must be reflected in `STATE.md` and `CHANGELOG.md` when the repo is active.

## How to Tune the Skill

1. Open `.grok/skills/loop-triage/SKILL.md`.
2. Make targeted changes based on daily findings.
3. Keep additions concise and actionable.

## Workflow

```bash
cp starters/minimal-loop/.grok/skills/loop-triage .grok/skills/
cp starters/minimal-loop/STATE.md.example STATE.md
```

Then customize `STATE.md` project name and run `/loop 1d`.
