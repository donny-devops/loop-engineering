# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |

## Reporting a Vulnerability

- Do not open a public issue for security vulnerabilities.
- Email: **donny@bizdevgurus.com**
- Subject: `[SECURITY] loop-engineering`
- Include:
  - Summary of the issue
  - Affected file(s)/component(s)
  - Reproduction steps or proof-of-concept
  - Suggested fix (optional)
- Response time: within 3 business days.
- Please do not disclose the issue publicly until it is fixed.

## Incident Workflow

1. Triage and validate report.
2. Prepare patched release branch.
3. Contact submitter for confirmation.
4. Publish advisory and release.
5. Credit reporter if desired.
